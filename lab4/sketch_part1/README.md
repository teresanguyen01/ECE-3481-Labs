### Part 1: Concurrency

Concurrency refers to a program’s ability to handle multiple tasks at once. On an Arduino, only one process runs at a time, but by rapidly switching between processes, it appears as though multiple tasks are happening simultaneously. On a system with multiple cores, two separate processes will run on two separate cores, so they actually run at the same time. In our lab, we need to create multiple processes, with each process having its own stack, a scheduler that switches between them using ```yield()``` and a timer, and each process gets a turn to run. So it’s like switching between processes very quickly to create the illusion that they are running at the same time.

According to the specification, the function ```process_select(unsigned int cursp)``` is called with interrupts disabled, where cursp is the current process’s stack pointer. If there is no current process, or if the current process has been terminated, cursp is 0. The function should return 0 if no process is ready to run, or the stack pointer of the next ready process otherwise. In code, we can plan this out as follows: if cursp != 0, then the current process is still running, so we save its stack pointer and put it back into the ready queue. If cursp == 0, then the process is terminated or does not exist, so we do not requeue it. Then we check the ready queue. If the ready queue is empty, we set the current process to NULL and 0 is returned. If it is not empty, this means we still have processes to run, so we dequeue the next process, set it as the current process, and return its stack pointer.

We also have to build the queue. The queue is a FIFO (first in, first out) structure, so we should use a linked list queue with a head and tail so that we can store our process structures in the queue. We need an enqueue function and a dequeue function, with both head and tail initially set to NULL. For enqueue, we first check if the queue is empty. If it is empty, we set both head and tail to the process pointer. If it is not empty, we set the current tail’s next pointer to the process and then update the tail to the new process. For dequeue, we first check if the queue is empty, and if it is, we return ```NULL```. Otherwise, we remove the head of the queue and update the head pointer. We also set the removed process’s next pointer to ```NULL```. If the new head becomes ```NULL```, then the queue is now empty, so we set the tail to``` NULL```.


Each process is represented by a ```struct process_state```, which contains a stack pointer sp and a next pointer linking to the next process in the queue. The global variable current_process keeps track of the currently running process. To create a process, ```process_create(void (*f)(void), int n)``` allocates a new ```process_t```, initializes its stack using process_init(f, n), inserts it into the ready queue, and returns -1 on failure or 0 on success. The function ```process_start()``` begins scheduling by setting current_process to NULL and calling ```process_begin()``` to start running processes. Context switching occurs when the timer triggers ```yield()```, which in turn calls ```process_select()``` to determine the next process to run.
